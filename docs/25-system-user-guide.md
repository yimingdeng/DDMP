# 玉米重点品种数字展示平台使用帮助文档

本文档面向平台日常使用人员，说明后台管理、移动端采集、营销发布、公开展示和线索跟进等模块的使用方法。文档重点回答三个问题：

1. 不同角色从哪里登录；
2. 登录后主要做什么；
3. 哪些内容需要审核、发布、回填和跟进。

相关功能基线：`ADM-AUTH-001`、`ADM-AUTH-002`、`COL-FIELD-001`、`COL-REVIEW-001`、`MKT-CENTER-001`、`MKT-PACKAGE-001`、`MKT-LINK-001`、`MKT-PUBLISH-001`、`MKT-DASHBOARD-001`、`FE-SHARE-001`、`FE-SITE-SORT-001`。

## 1. 系统入口总览

| 入口 | 地址 | 主要用户 | 主要用途 |
|---|---|---|---|
| 公开展示首页 | `/` | 客户、经销商、农户、销售人员 | 查看品种、示范点、联系方式、咨询入口 |
| 品种详情页 | `/varieties/<品种>/` | 客户、销售、经销商 | 查看单一品种核心卖点、图片、视频、示范表现 |
| 看示范 | `/sites/` | 客户、销售、经销商 | 地图/列表查看示范点，按距离查看附近示范 |
| 我的示范 | `/collection/` | 示范点负责人、采集人员 | 登录后提交田间阶段数据、照片、视频 |
| 申请成为示范户 | `/collection/apply/` | 农户、合作社、潜在示范户 | 提交示范申请，等待后台电话沟通和审核 |
| 我要发布 / 营销发布中心 | `/marketing/` | 运营、销售、经销商、示范负责人 | 生成营销素材、专属链接、二维码、海报、文案 |
| 后台管理 | `/admin/` | 系统管理员、运营审核、区域管理员 | 管理品种、示范点、媒体、采集审核、线索和统计 |

正式公网访问时，入口通常使用：

```text
https://bzb889.originseed.com.cn/
```

本地开发测试时，入口通常使用：

```text
http://127.0.0.1:8000/
```

## 2. 用户角色与权限说明

### 2.1 系统管理员

系统管理员负责整个平台的基础配置、账号权限、数据维护和发布审核。

主要工作：

- 创建和维护后台账号；
- 配置用户组、权限和手机号；
- 管理品种、卖点、示范点、媒体素材；
- 审核田间采集内容；
- 审核示范申请；
- 管理区域联系人；
- 查看咨询线索、访问统计和营销效果；
- 维护站点配置、主题色、分享图、客服电话等。

常用入口：

```text
/admin/
```

注意事项：

- 不要把超级管理员账号交给普通业务人员使用；
- 修改账号时必须维护手机号；
- 正式环境不要上传测试图片、测试电话和无效客户信息；
- 下线内容前要确认是否仍在营销传播中使用。

### 2.2 后台运营 / 内容审核人员

运营人员负责把田间采集内容整理成可公开展示和可营销传播的内容。

主要工作：

- 审核示范点负责人提交的阶段表现；
- 检查照片、视频是否清晰、合规；
- 修改公开摘要和关键表现标签；
- 发布阶段表现；
- 生成或刷新营销素材；
- 回填微信、视频号、抖音等发布记录；
- 查看营销周报和效果看板。

常用入口：

```text
/admin/
/marketing/
```

建议流程：

1. 在后台查看待审核采集记录；
2. 检查阶段、照片、视频、异常天气、倒伏、病害等信息；
3. 确认公开摘要不包含内部评价和敏感信息；
4. 审核通过并发布；
5. 进入营销发布中心生成素材包；
6. 提供给销售、经销商或公司运营发布；
7. 发布后登记外部发布记录和效果数据。

### 2.3 区域管理员 / 区域经理

区域管理员负责本区域示范户、示范点和客户线索跟进。

主要工作：

- 查看本区域示范申请；
- 电话沟通申请人；
- 协助后台创建或完善示范点；
- 跟进本区域客户咨询；
- 维护区域服务经理联系方式；
- 协助销售和经销商使用营销素材。

常用入口：

```text
/admin/
/marketing/
```

注意事项：

- 区域联系人信息会在公开页面展示，应保证姓名和电话准确；
- 客户咨询提交后应及时电话跟进；
- 对于申请成为示范户的信息，系统不展示进度查询，主要由后台人员电话沟通。

### 2.4 示范点负责人 / 采集人员

示范点负责人主要使用手机端提交田间阶段数据、照片和视频。

常用入口：

```text
/collection/
```

主要工作：

- 登录“我的示范”；
- 查看自己负责的示范点；
- 按当前阶段提交田间表现；
- 拍照、摄像并上传；
- 填写或选择标准化数据；
- 查看自己提交记录的审核状态。

采集阶段包括：

- 播种 / 出苗；
- 拔节；
- 抽雄吐丝；
- 灌浆；
- 成熟；
- 收获测产。

采集建议：

- 优先使用系统选择项，不要写大段自由文字；
- 照片尽量现场直接拍摄；
- 每个阶段至少上传能说明问题的田间照片；
- 如果有倒伏、病害、异常天气，应按标准项记录；
- 视频建议拍摄 10—30 秒，画面稳定，先拍整体再拍特写；
- 数据不确定时不要夸大，宁可留给后台审核人员补充。

注意事项：

- 采集端不是公开展示端，提交后需要后台审核；
- 未审核内容不会直接对客户公开；
- 密码至少 6 位即可，不要求复杂密码；
- 手机登录会尽量保留上次填写账号，方便下次进入。

### 2.5 营销发布人员

营销发布人员包括公司运营、销售人员、经销商和示范负责人中承担传播任务的人员。

常用入口：

```text
/marketing/
```

主要工作：

- 选择已审核的营销素材包；
- 选择传播渠道；
- 生成专属链接和二维码；
- 复制朋友圈、微信群、客户私聊、视频号、抖音文案；
- 保存海报、短视频封面和二维码；
- 下载一键营销 ZIP 包；
- 发布到微信、视频号、抖音或线下物料；
- 发布后回填发布记录。

传播渠道说明：

| 渠道 | 使用场景 |
|---|---|
| 微信朋友圈 | 海报传播、区域热度扩散、看田邀请 |
| 微信群 | 快速触达经销商、零售商、种植大户 |
| 微信视频号 | 发布田间短视频、测产现场、看田预告 |
| 抖音短视频 | 扩大曝光、吸引新客户 |
| 抖音直播 | 看田直播、测产直播、会议直播 |
| 田间二维码 | 线下展牌、海报、观摩会扫码 |

重要原则：

- 渠道表示“发到哪里”；
- 销售、经销商、区域经理属于“人员身份”，后续通过后台用户组或绑定身份识别；
- 每次发布都应使用系统生成的专属链接或二维码；
- 不要手工复制普通页面链接替代专属链接，否则统计和线索归因会不准确。

## 3. 公开展示端使用说明

公开展示端面向客户，不需要登录。

### 3.1 首页

首页主要展示：

- 主推品种；
- 首屏图片或视频；
- 品种核心卖点；
- 看详情；
- 看示范；
- 我要咨询；
- 联系我们；
- 我的示范；
- 我要发布。

移动端展示以图片、视频和短句为主，避免大段文字。

### 3.2 品种详情

品种详情用于向客户说明：

- 这是什么品种；
- 适合什么区域；
- 核心优势是什么；
- 有哪些真实示范表现；
- 如何咨询、看田或联系销售。

本地 MP4 视频点击后会尽量最大化播放。外部视频链接会跳转到对应平台。

### 3.3 看示范

客户点击“看示范”后进入示范点页面。

主要能力：

- 默认地图模式；
- 默认展示黄淮海等重点区域；
- 请求定位后优先显示附近示范点；
- 地图使用图钉标识示范点；
- 支持查看示范点详情；
- 示范点详情按生育阶段倒序展示公开阶段表现。

注意事项：

- 如果用户拒绝定位，系统按后台维护的展示顺序兜底；
- 公开页面不展示未审核照片和内部备注；
- 阶段照片优先展示，文字说明尽量简洁。

### 3.4 我要咨询

客户可以在公开页面提交咨询信息。

建议填写：

- 姓名；
- 电话；
- 所在地区；
- 身份；
- 感兴趣事项；
- 备注。

提交后系统会提示已提交，并记录来源渠道、营销素材、推广人和追踪链接，方便后续跟进。

## 4. 后台管理使用说明

后台管理面向管理员、运营审核和区域管理员。

入口：

```text
/admin/
```

### 4.1 登录

可使用以下任一方式作为登录名：

- 账号；
- 邮箱；
- 手机号。

登录账号必须拥有后台权限；没有权限的人员不应进入后台后再看到“无权限”，而应在登录时被拦截。

### 4.2 用户和权限管理

系统管理员可维护：

- 用户；
- 用户组；
- 权限；
- 手机号；
- 姓名；
- 是否启用；
- 是否工作人员；
- 是否超级管理员。

建议按角色建组：

- 系统管理员；
- 内容运营；
- 区域管理员；
- 采集负责人；
- 营销发布人员；
- 销售人员；
- 经销商。

### 4.3 站点配置

可维护：

- 网站名称；
- 公司名称；
- Logo；
- 默认分享标题；
- 默认分享描述；
- 默认分享图；
- 主题色；
- 客服电话；
- 高德地图 Key；
- 公开域名。

主题色当前支持：

- 系统基础色调；
- 紫色 + 黄色产品营销色调。

### 4.4 品种管理

品种管理用于维护单一主推品种及其展示内容。

常见字段：

- 品种名称；
- URL 标识；
- 适宜区域；
- 品种简介；
- 定位语；
- 是否推荐；
- 发布状态；
- 展示顺序。

操作建议：

- 主推品种设为推荐；
- URL 标识不要频繁修改；
- 简介面向客户，避免内部术语；
- 未准备好的品种保持草稿状态。

### 4.5 卖点管理

卖点用于将品种优势拆成多个简洁模块。

建议写法：

- 标题短；
- 描述短；
- 有图片或视频时优先配置；
- 移动端展示以紧凑卡片为主；
- 归档的卖点不再对外展示。

### 4.6 图片与视频管理

媒体素材支持：

- 普通图片；
- 外部视频链接；
- 本地 MP4 视频；
- 视频封面。

注意事项：

- 图片用于封面、画廊、卖点和示范展示；
- 本地视频必须上传 MP4 文件；
- 本地视频建议使用 H.264 + AAC 编码；
- 视频设为封面时必须配置视频封面图；
- 替代文本主要用于图片无法显示、搜索和无障碍阅读；
- 不要把内部资料、客户手机号、未审核内容放到公开媒体素材中。

### 4.7 示范点管理

示范点用于承载田间展示和采集数据。

常见字段：

- 示范点名称；
- 所属品种；
- 区域、省、市、县、乡镇、详细地址；
- 当前阶段；
- 经纬度；
- 负责人；
- 展示顺序；
- 发布状态。

操作建议：

- 示范点名称可由“县 + 乡镇”自动生成，管理员后续可修改；
- 经纬度应尽量准确，方便地图展示和距离计算；
- 当前阶段由公开阶段自动推进，避免手工倒退；
- 展示顺序用于定位失败时兜底排序。

### 4.8 示范申请审核

客户或农户可提交“申请成为示范户”。

后台处理流程：

1. 查看新申请；
2. 电话沟通确认情况；
3. 判断是否适合作为示范点；
4. 区域管理员或总部审核；
5. 审核通过后创建或关联示范点；
6. 为负责人创建采集账号；
7. 通知负责人登录“我的示范”。

系统不提供公开进度查询，具体沟通由后台人员电话完成。

### 4.9 采集审核与发布

示范点负责人提交阶段表现后，后台进行审核。

审核重点：

- 阶段是否正确；
- 照片是否清晰；
- 视频是否可播放；
- 数据是否合理；
- 是否有异常天气、倒伏、病害等关键指标；
- 公开摘要是否适合客户阅读；
- 是否包含内部备注或敏感信息。

审核通过后，阶段表现才会进入公开展示和营销素材生成流程。

### 4.10 区域联系人与咨询线索

区域联系人用于公开页面展示“区域服务经理”。

咨询线索用于记录客户提交的信息。

线索跟进建议：

1. 每天查看新增咨询；
2. 按区域分配给区域经理；
3. 电话联系客户；
4. 记录跟进结果；
5. 对有意向客户安排看田、试种或经销沟通。

## 5. 移动端采集使用说明

入口：

```text
/collection/
```

### 5.1 登录

示范点负责人使用分配的账号登录。

登录后进入自己的示范点列表或采集任务页。

### 5.2 提交阶段数据

采集页面会把当前需要采集的阶段重点展示。

操作步骤：

1. 打开“我的示范”；
2. 输入账号和密码；
3. 选择示范点；
4. 查看当前重点阶段；
5. 点击进入采集；
6. 按页面选项填写数据；
7. 拍照或摄像上传；
8. 保存并提交。

### 5.3 拍照和摄像

手机端支持：

- 直接调用相机拍照；
- 一次上传多张照片；
- 直接调用摄像；
- 上传多个现场视频。

建议：

- 照片先拍整体，再拍近景；
- 视频控制在 10—30 秒；
- 拍摄时尽量横向或稳定移动；
- 不要上传无关人物、车牌和客户隐私。

### 5.4 自动计算

系统会尽量减少手工输入。

示例：

- 输入播种密度和行距后，自动计算株距；
- 自动保留一位小数；
- 阶段指标优先用选择项；
- 能由已有数据推导的内容尽量自动计算。

### 5.5 退回首页

采集页面提供返回首页功能，方便负责人从采集端回到公开展示首页。

## 6. 营销发布中心使用说明

入口：

```text
/marketing/
```

### 6.1 登录和权限

营销发布中心面向：

- 管理员；
- 运营人员；
- 销售人员；
- 经销商；
- 示范负责人。

没有营销权限的账号不应看到无权限菜单。

### 6.2 选择素材包

进入营销发布中心后，可查看已审核可发布的营销素材包。

筛选方式：

- 阶段；
- 区域；
- 状态；
- 关键词。

素材包通常来自已经公开的示范阶段表现。

### 6.3 生成专属传播链接

操作步骤：

1. 打开素材包；
2. 在第一步选择传播渠道；
3. 点击生成专属链接和二维码；
4. 系统显示专属链接；
5. 系统显示二维码；
6. 后续文案、海报和 ZIP 包都会使用这个专属链接。

专属链接示例：

```text
https://bzb889.originseed.com.cn/q/go/<token>/?v=<版本号>
```

说明：

- `<token>` 用于识别营销素材、渠道和推广人；
- `v` 用于降低微信分享卡片缓存影响；
- `v` 不影响来源统计；
- 客户点击专属链接后，会进入带分享配置的中转页，再自动打开示范阶段页面。

### 6.4 复制文案

系统提供多类文案：

- 朋友圈文案；
- 客户私聊文案；
- 微信群文案；
- 视频号标题与文案；
- 抖音标题和话题；
- 短视频脚本。

使用建议：

- 朋友圈文案要短；
- 微信群文案要有链接；
- 客户私聊文案可稍微口语化；
- 视频号和抖音一条内容只讲一个重点；
- 不要夸大宣传，不要写“保证高产”等绝对承诺。

### 6.5 保存海报和二维码

素材包可生成：

- 朋友圈海报；
- 短视频封面；
- 经销商 / 推广人专属海报；
- 观摩会 / 看田邀请海报；
- 本周重点推荐海报；
- 专属二维码。

用途：

- 发朋友圈；
- 发微信群；
- 放到线下海报；
- 用于观摩会现场扫码；
- 发给经销商转发。

### 6.6 下载一键营销包

ZIP 包通常包含：

- 海报；
- 短视频封面；
- 各渠道文案；
- 短视频脚本；
- 专属链接；
- 专属二维码；
- 场景海报。

适合发给销售、经销商或运营同事统一使用。

### 6.7 回填微信 / 抖音发布记录

发布后应回填发布记录。

可登记：

- 发布渠道；
- 发布状态；
- 发布标题；
- 发布账号；
- 外部链接；
- 发布时间；
- 浏览 / 播放量；
- 点赞量；
- 评论量；
- 转发量；
- 备注。

规则：

- 朋友圈和微信群可以只登记发布动作和时间；
- 视频号、抖音短视频、抖音直播需要填写外部链接，方便客户页展示视频入口；
- 草稿、归档或无链接的视频记录不会公开展示。

### 6.8 营销统计和周报

## 7. 微信、二维码和分享说明

### 7.1 直接链接、扫码和转发

系统希望三种入口保持一致：

```text
直接点击专属链接
扫码专属二维码
微信内继续转发
```

当前做法：

- 专属链接先打开带分享配置的中转页；
- 中转页包含标题、描述、封面和专属 URL；
- 浏览器自动跳到真正阶段展示页；
- 阶段展示页继续保留微信 JS-SDK 分享配置。

### 7.2 微信分享生效条件

要让微信卡片稳定显示标题、封面和描述，需要：

- 正式 HTTPS 域名；
- 认证公众号或具备 JS-SDK 权限的公众号；
- 正确配置 AppID 和 AppSecret；
- 微信公众平台配置 JS 接口安全域名；
- 服务器可访问微信接口；
- 微信未继续使用旧缓存。

如果微信分享卡片没有变化，应检查：

1. 链接是否为 `/q/go/<token>/?v=...`；
2. 页面源代码是否有 `og:title`、`og:description`、`og:image`；
3. 是否启用了微信 JS-SDK；
4. 公众号后台 JS 安全域名是否正确；
5. 微信是否缓存了旧链接。

## 9. 常见问题

### 9.1 为什么后台能看到内容，前台看不到？

常见原因：

- 内容仍是草稿；
- 品种未发布；
- 示范点未发布；
- 阶段表现未审核；
- 媒体素材未发布；
- 营销素材包未标记为可发布。

### 9.2 为什么地图没有按最近示范点展示？

可能原因：

- 用户拒绝定位；
- 浏览器没有定位权限；
- 示范点没有维护经纬度；
- 手机网络定位不准确；
- 当前环境不是 HTTPS。

### 9.3 为什么微信分享卡片不对？

可能原因：

- 微信缓存旧链接；
- 没有使用专属链接；
- 公众号 JS-SDK 未配置；
- JS 接口安全域名不正确；
- 分享图太小或不可访问；
- 页面不是 HTTPS。

### 9.4 为什么咨询没有归因到推广人？

可能原因：

- 客户不是通过专属链接或二维码进入；
- 使用了普通页面链接；
- 浏览器禁用 Cookie；
- 链接 token 已停用；
- 客户隔了很久才提交，来源会话已失效。

### 9.5 为什么视频不能播放或不能全屏？

可能原因：

- 上传文件不是有效 MP4；
- 编码不是常见 H.264 + AAC；
- 文件过大；
- 浏览器不支持当前视频；
- 手机系统限制自动播放。

公开前端本地视频点击后会尽量最大化播放；如果浏览器不支持原生全屏，会使用页面内全屏弹层兜底。

## 10. 日常使用建议

### 10.1 管理员每日检查

- 新咨询线索；
- 新示范申请；
- 待审核采集记录；
- 异常天气和倒伏病害记录；
- 公开页面是否正常；
- 营销素材是否生成成功。

### 10.2 运营每周工作

- 筛选本周重点示范点；
- 审核阶段表现；
- 生成营销素材包；
- 输出朋友圈海报、微信群文案、视频号脚本、抖音标题；
- 生成营销周报；
- 汇总线索来源和转化情况。

### 10.3 销售和经销商发布建议

- 每次发布都使用专属链接或二维码；
- 不要只发大段文字；
- 优先发真实田间图片和短视频；
- 一条内容只讲一个重点；
- 发布后及时回填记录；
- 对提交咨询的客户尽快电话跟进。

### 10.4 示范点负责人采集建议

- 关键阶段及时采集；
- 尽量现场拍摄；
- 数据真实，不夸大；
- 发现异常及时记录；
- 照片和视频比长文字更重要；
- 提交后等待后台审核。

## 11. 权限和数据安全提醒

- 不要共享管理员账号；
- 不要在公开页面展示客户手机号和内部备注；
- 不要发布未审核田间数据；
- 不要在海报和视频中过度承诺产量；
- 正式环境不要使用测试数据；
- 删除或下线内容前，先确认是否已被二维码、海报、朋友圈或微信群传播；
- 导出数据时注意客户隐私和公司内部资料保护。

# 第三阶段 Sprint 12–14：营销素材、传播追踪与线索回流

## 1. 建设目标

Sprint 12–14 在现有“示范采集 → 两级审核 → 对外公开 → 客户咨询”基础上，增加第一版营销传播闭环：

```text
示范负责人提交阶段表现
        ↓
区域审核、总部审核并发布公开快照
        ↓
自动生成阶段营销素材包
        ↓
微信海报、微信群文案、视频号/抖音文案、短视频脚本和二维码
        ↓
销售、经销商或示范负责人传播
        ↓
客户进入对应阶段页面并提交咨询
        ↓
后台记录渠道、素材、推广人和咨询意向
```

本阶段复用现有 Django 单体、`PublishedObservation` 公开快照、渠道二维码、微信分享、咨询线索和访问统计，不建设第二套内容后台。

## 2. Sprint 12：阶段传播页面和追踪链接

对应需求：`MKT-CONTENT-001`、`MKT-LINK-001`。

### 2.1 独立阶段传播页面

- 每个已公开阶段快照可以生成独立、稳定的传播地址。
- 地址包含不可预测的内容 UUID，不使用示范点名称、客户信息或内部账号作为传播标识。
- 阶段页面优先展示审核时已经存在的现场图片和视频，随后展示公开摘要和结构化数据。
- 页面自动生成微信分享标题、摘要和封面。
- 示范点详情中的每条公开阶段增加“查看并分享本阶段表现”入口。
- 未公开的采集记录不能生成传播页面。
- 素材包停用、示范点下线或品种下线后，带内容 UUID 的传播页面返回 404。

页面地址：

```text
/sites/<示范点标识>/stages/<阶段>/
/sites/<示范点标识>/stages/<阶段>/<内容UUID>/
```

第一种地址显示该阶段最新公开快照；第二种地址固定对应具体公开版本，适合海报、二维码和历史传播。

### 2.2 传播追踪链接

`TrackedLink` 将以下信息绑定到一个随机访问标识：

- 营销素材包。
- 渠道来源代码。
- 可选推广身份。
- 用途说明。
- 启用状态。
- 累计访问次数和最近访问时间。

访问入口：

```text
/q/go/<访问UUID>/
```

系统验证链接和公开内容有效后，记录访问并跳转到固定的阶段传播页面。链接停用或内容不可用时返回平台首页并显示友好提示。

### 2.3 阶段二维码

现有 `ChannelQRCode` 增加“示范阶段表现”目标类型：

- 可以选择具体 `PublishedObservation` 公开快照。
- 二维码保留受控 `source` 来源代码。
- 跳转到对应内容 UUID 页面，不会因后续发布新版本而改变原海报内容。
- 原有首页、品种、示范点和咨询二维码继续有效。

## 3. Sprint 13：极简反馈和传播归因

对应需求：`MKT-LEAD-001`、`MKT-ATTR-001`。

### 3.1 客户身份

公开咨询表单增加必填“您的身份”：

- 经销商。
- 农户。
- 种植大户。
- 合作社。
- 零售商。
- 其他。

### 3.2 客户意向

咨询类型扩展为：

- 产品咨询。
- 预约看田。
- 申请试种。
- 代理合作。
- 参加观摩会。
- 提交反馈。

姓名、电话、区域、客户身份和咨询类型由服务端校验；隐私同意、防重复提交、隐藏反机器人字段和频率限制继续生效。

### 3.3 来源回流

客户通过追踪链接或阶段二维码进入后，会话保存：

- `source_code`：渠道来源。
- `marketing_package`：具体营销素材包。
- `promotion_identity`：销售、经销商或示范负责人。
- `tracked_link`：具体传播链接。
- `source_path`：首次落地页面。

`VisitEvent` 和 `Inquiry` 同时保存以上归因关系，因此可以区分“产生访问”和“最终产生咨询”的具体内容与推广人。

公开链接只使用随机 UUID；推广人姓名、手机号、账号和内部备注不会写入 URL。

## 4. Sprint 14：微信海报、视频封面和传播文案

对应需求：`MKT-POSTER-001`、`MKT-COPY-001`。

### 4.1 自动生成时机

总部发布 `PublishedObservation` 后，系统在数据库事务成功提交后自动：

1. 创建 `MarketingPackage`。
2. 根据阶段公开数据提取最多三个核心标签。
3. 生成默认朋友圈追踪链接。
4. 生成朋友圈海报。
5. 生成竖版短视频封面。
6. 生成微信、视频号、抖音文案和短视频脚本。

生成失败不会回滚已经审核通过的公开数据；系统记录错误，管理员可以在后台或通过管理命令重新生成。

### 4.2 朋友圈海报

海报尺寸为 `1080 × 1440`，内容包括：

- 品种名称。
- 示范点名称。
- 当前生育阶段。
- 最多三个核心表现标签。
- 阶段第一张公开田间照片；没有照片时使用产品紫色品牌背景。
- 公司 Logo；未配置 Logo 时不显示。
- “扫码查看完整表现”二维码。

营销图片读取“站点配置 → 系统主色调”：支持系统基本绿色以及“紫色主背景 + 黄色标题/强调 + 白色信息区”两套规则。实拍照片不套色、不改变田间表现。该规则仅用于公开营销内容，不改变业务采集端和后台管理端。切换主题后，需要重新生成已有海报和封面才能更新历史图片。

二维码指向默认 `wechat_moments` 追踪链接，扫码后进入该公开快照对应的阶段页面。

### 4.3 短视频封面

- 尺寸为 `1080 × 1440`。
- 使用阶段田间主图作为主体，配合紫色渐变底、黄色标题和品牌装饰线。
- 展示品种、示范点、生育阶段和公开摘要。
- 输出 JPEG，供视频号和抖音人工发布时使用。

### 4.4 自动文案

每个素材包生成：

- 微信群文案。
- 视频号标题。
- 视频号文案。
- 抖音标题。
- 抖音话题。
- 短视频口播脚本。

首版采用确定性模板，不依赖外部大模型接口。后台可以直接编辑生成后的字段并保存。

注意：后台“重新生成所选素材”会重新应用默认模板并覆盖当前文案。需要保留人工文案时，应只保存编辑结果，不执行重新生成操作。将“仅重新生成图片”和“恢复默认文案”拆分为两个操作，安排在后续优化。

## 5. 数据模型

### `MarketingPackage`

- 与一个 `PublishedObservation` 一对一关联。
- 保存公开内容 UUID、状态、核心标签、各渠道文案、海报和视频封面。
- 每个公开快照版本拥有独立素材包，历史传播不会自动切换到新版本。

### `PromotionIdentity`

- 推广类型：销售人员、经销商、示范点负责人。
- 保存内部代码、区域、有效状态和随机公开 UUID。
- 不保存或公开客户手机号。

### `TrackedLink`

- 关联营销素材包、渠道和可选推广身份。
- 保存点击次数和最近点击时间。
- 停用后不再跳转到营销内容。

### `ChannelQRCode`

- 新增 `published_observation` 目标。
- 保留原二维码扫描统计和公网地址生成规则。

### `Inquiry` 与 `VisitEvent`

- 新增营销素材、推广身份和追踪链接关联。
- `Inquiry` 增加客户身份和两类新意向。

## 6. 管理后台操作

### 6.1 查看营销素材

1. 登录 `/admin/`。
2. 打开“传播与二维码 → 营销素材包”。
3. 按状态、阶段或区域筛选。
4. 打开素材后查看：
   - 传播标题和核心标签。
   - 微信群文案。
   - 视频号文案。
   - 抖音文案和话题。
   - 短视频脚本。
   - 朋友圈海报和视频封面预览。
   - 阶段传播页面。
5. 文案需要调整时直接编辑并保存。

### 6.2 重新生成素材

1. 在营销素材包列表勾选记录。
2. 选择“重新生成所选素材的文案、海报和封面”。
3. 执行后检查成功或失败提示。

该操作覆盖默认文案、海报和视频封面，应在确认无需保留人工修改后执行。

### 6.3 配置推广身份和专属链接

1. 打开“传播与二维码 → 推广身份”。
2. 创建销售、经销商或示范负责人身份。
3. 打开“传播与二维码 → 传播追踪链接”。
4. 选择素材包、来源代码和推广人。
5. 保存后复制传播短链接。

建议来源代码：

| 场景 | 来源代码 |
|---|---|
| 微信朋友圈 | `wechat_moments` |
| 微信群 | `wechat_group` |
| 视频号 | `wechat_channels` |
| 抖音短视频 | `douyin_video` |
| 抖音直播 | `douyin_live` |
| 田间牌二维码 | `field_qrcode` |

说明：传播渠道只表示发布位置或入口来源；销售、经销商等属于发布人员身份，后续由后台用户组或绑定身份识别，不再混入渠道列表。

### 6.4 创建阶段二维码

1. 打开“传播与二维码 → 渠道二维码”。
2. 目标类型选择“示范阶段表现”。
3. 选择已经公开的阶段快照。
4. 设置唯一来源代码和用途说明。
5. 保存后预览或下载 PNG。

## 7. 历史数据补生成

升级数据库后执行：

```powershell
cd D:\DDMP
.\.venv\Scripts\python.exe manage.py migrate
.\.venv\Scripts\python.exe manage.py generate_marketing_materials --missing-only
```

运行环境使用：

```powershell
cd D:\DDMP-RUNTIME\app\current
D:\DDMP-RUNTIME\venv\Scripts\python.exe manage.py migrate
D:\DDMP-RUNTIME\venv\Scripts\python.exe manage.py generate_marketing_materials --missing-only
D:\DDMP-RUNTIME\venv\Scripts\python.exe manage.py collectstatic --noinput
```

`--missing-only` 只处理缺少海报或视频封面的素材包，可以安全重复执行。生成文件保存在运行环境的 `MEDIA_ROOT\marketing\` 下，数据库和媒体目录必须一起备份。

## 9. 当前边界与后续 Sprint

Sprint 12–14 提供“生成素材、人工传播、回流归因”的第一版闭环。以下能力已经在 Sprint 15 完成：

- 手机端营销发布中心。
- 素材待审核、可发布、已登记发布和已停用状态。
- 一键复制微信群、视频号、抖音文案和短视频脚本。
- 销售、经销商和示范负责人账号绑定与专属追踪链接。
- 包含海报、封面、文案、脚本、链接和二维码的 ZIP 转发包。

仍未包含：

- 批量创建推广链接。
- 视频号或抖音自动发布。
- 视频发布登记和外部播放量回填。
- 营销转化看板、排行榜和周报。
- 自动合成带配音、字幕的完整短视频。
- 微信小程序、抖音小程序和视频/直播挂载。

剩余功能进入 Sprint 16 以后实施。Sprint 15 详细说明见 `20-sprint-15-marketing-publishing-center.md`。

# 第二阶段 Sprint 10–11：异常、审核与公开

## 1. Sprint 10：田间异常上报

负责人在示范点任务页点击“上报田间异常”，填写：

- 发生阶段。
- 异常类型：缺苗、虫害、病害、干旱、涝害、倒伏/倒折、恶劣天气、管理异常或其他。
- 严重程度：轻微、中等、严重。
- 发生日期、情况说明和建议措施。
- 至少一张现场照片，可连续拍照或从相册多选。

异常记录出现在负责人示范点页面、后台异常列表以及对应阶段的审核详情中。后台可以将异常标记为已处理，原始上报人、时间和照片保留。

## 2. Sprint 11：两级审核

状态流程：

```text
草稿 → 已提交 → 区域审核通过 → 总部审核通过 → 已公开
            ↘ 退回修改 ←───────────────↙
                 → 负责人修改并重新提交
```

权限规则：

- 负责人只能编辑草稿和已退回记录。
- 已提交、区域通过、总部通过和已公开记录在负责人端只读。
- 区域审核人只能查看并审核其负责区域的示范点。
- 总部审核人可以查看全部区域，但只能审核区域已经通过的数据。
- 退回必须填写原因。
- 发布必须填写对外展示摘要。
- 所有保存、提交、通过、退回和发布动作记录账号、时间、状态和意见，普通用户不能修改历史。

## 3. 审核账号配置

1. 后台“认证和授权 → 用户”创建个人账号。
2. 后台“田间采集 → 采集审核人”新增审核角色。
3. 区域审核人必须选择负责区域；总部审核人不选择区域。
4. 审核人使用负责人端登录地址登录，顶部显示“审核工作台”。

审核入口：`/collection/review/`。

## 4. 内部数据与公开数据隔离

总部点击发布时，系统创建 `PublishedObservation` 公开快照：

- 原始 `Observation` 不被公开文案覆盖。
- 快照保存版本号、公开数据、公开摘要、发布人和发布时间。
- 种子批次、底肥方案和负责人内部评价不进入公开快照。
- 示范点公开详情页只展示已经发布的最新阶段快照。
- 公开页面可展示该阶段审核通过的现场照片和视频。

## 5. 验收重点

- [ ] 未分配示范点的负责人不能上报该示范点异常。
- [ ] 异常没有现场照片时不能提交。
- [ ] 区域审核人修改 URL 仍不能查看其他区域记录。
- [ ] 总部不能跳过区域审核直接通过。
- [ ] 退回原因在负责人端历史中可查看。
- [ ] 退回记录可以修改并重新提交。
- [ ] 发布后公开页出现摘要和阶段数据。
- [ ] 内部批次、底肥和内部评价不出现在公开页面。

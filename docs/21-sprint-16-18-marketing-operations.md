# 第三阶段 Sprint 16–18：外部发布、营销看板与运营周报

## 1. 建设目标

在 Sprint 12–15 已完成“生成素材 → 一键营销 → 专属链接 → 线索回流”的基础上，Sprint 16–18 继续补齐运营闭环：

```text
一键营销素材
    ↓
人工发布到视频号 / 抖音 / 朋友圈 / 经销商渠道
    ↓
回填外部发布链接和平台指标
    ↓
客户在阶段页看到已发布视频入口
    ↓
访问、咨询、推广人、外部播放数据进入看板
    ↓
生成营销周报，指导下一周发布和跟进
```

本阶段仍然坚持 Django 单体、模板页面和人工审核发布，不接入微信/抖音自动发布接口，也不开发小程序挂载。

## 2. Sprint 16：微信/抖音发布记录与短视频选题

对应需求：`MKT-PUBLISH-001`、`MKT-TOPIC-001`。

### 2.1 外部发布记录

新增“外部发布记录”模型，关联到一个营销素材包，字段包括：

- 发布渠道：微信朋友圈、微信群、微信视频号、抖音短视频、抖音直播、田间二维码。
- 发布状态：待发布、已发布、已归档。
- 发布标题、发布账号、外部链接、发布时间。
- 播放/浏览量、点赞量、评论量、转发量。
- 备注、登记人、登记时间和更新时间。

使用方式：

1. 打开 `/marketing/`。
2. 进入某个“一键营销”素材。
3. 在“回填微信/抖音发布记录”区域填写发布时间；视频号、抖音等有公开链接的平台继续填写外部发布链接和指标。
4. 保存后后台和统计看板可查看。
5. 只有“已发布 + 有外部链接”的记录会展示到客户阶段页。

### 2.2 公开阶段页展示外部视频

示范阶段传播页会新增“已发布短视频”模块，展示：

- 发布渠道。
- 发布标题。
- 发布账号。
- 播放/浏览量。
- 外部跳转链接。

该模块用于让客户从品种阶段页继续查看视频号或抖音内容；草稿、归档和未填写链接的记录不会公开。

### 2.3 短视频选题拆分

每个营销素材包会生成多条“单重点”短视频选题：

- 标题。
- 单一重点。
- 口播脚本。
- 排序和启用状态。

营销发布中心详情页展示选题并支持一键复制脚本；后台可编辑标题、重点、脚本和排序。

## 3. Sprint 17：营销转化看板

对应需求：`MKT-DASHBOARD-001`。

新增入口：

```text
/marketing/stats/
```

该页面面向管理员和运营账号，普通推广账号不能查看全局数据。

看板展示最近 30 天：

- 营销访问量。
- 独立访客数。
- 回流咨询数。
- 外部发布数。
- 外部平台播放/浏览、点赞、评论和转发。
- 示范点访问排行。
- 示范点咨询排行。
- 来源渠道排行。
- 推广人线索排行。
- 最近外部发布记录。

看板数据来自现有 `VisitEvent`、`Inquiry`、`TrackedLink` 和新增 `ExternalPublication`，不额外引入 BI 或数据仓库。

## 4. Sprint 18：营销传播周报

对应需求：`MKT-WEEKLY-001`。

新增入口：

```text
/marketing/weekly/
```

系统按最近 7 天自动生成一份周报，内容包括：

- 本周营销传播访问量、独立访客和咨询数量。
- 外部视频/内容发布数量。
- 人工回填播放/浏览量。
- 主要来源渠道。
- 热门示范点。
- 下一步建议动作。

周报会保存为“营销周报”记录，管理员可在后台修改、归档，也可以在营销发布中心复制摘要和建议动作，用于周会、微信群或公众号选题准备。

## 5. 后台管理入口

后台“传播与二维码”相关模块新增：

- 短视频选题。
- 外部发布记录。
- 营销周报。

营销素材包编辑页内同时提供短视频选题和外部发布记录的内联维护能力。

## 6. 权限边界

- 未登录用户不能访问营销发布中心、看板和周报。
- 没有营销权限、也没有绑定推广身份的账号访问 `/marketing/` 时会退出当前登录态，并返回营销发布中心登录页，方便更换账号。
- 普通推广账号可以登记自己可见素材的外部发布记录，但不能查看全局统计看板和周报。
- 全局统计看板和营销周报仅限超级管理员或具备营销素材管理权限的运营账号访问。
- 公开客户页只展示已发布且带外部链接的外部发布记录。

## 9. 后续边界

尚未纳入本阶段：

- 微信或抖音自动发布。
- 抖音小程序短视频/直播自主挂载。
- 自动抓取抖音或视频号播放量。
- 自动合成带配音、字幕和转场的完整短视频。

这些能力需要外部平台认证、审核和接口权限，建议在平台运营流程稳定后单独规划。

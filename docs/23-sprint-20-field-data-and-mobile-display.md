# 第三阶段 Sprint 20：采集数据标准化与移动展示优化

## 1. 建设目标

本 Sprint 继续完善“数据采集和展示”能力，目标是让负责人更容易采集、让客户更快看懂示范表现：

- 示范点阶段数据更标准化。
- 异常天气、倒伏、病害等重点指标强化。
- 阶段照片自动优先展示。
- 示范点公开展示默认按距离呈现，弱化排序操作。
- 移动端页面继续压缩、提速和紧凑展示。

对应需求：`COL-DATA-STD-001`、`FE-STAGE-DISPLAY-001`、`FE-SITE-SORT-001`、`FE-MOBILE-PERF-001`。

## 2. 阶段采集数据标准化

采集表单继续优先使用：

- 日期。
- 数字。
- 单选。
- 多选。
- 自动计算结果。

本 Sprint 新增或强化的标准字段包括：

| 指标 | 说明 |
|---|---|
| 异常天气 | 高温、干旱、涝害、风灾、冰雹等 |
| 天气影响程度 | 无明显影响、轻微、中等、严重 |
| 倒伏等级 | 无倒伏、轻微倒伏、中等倒伏、严重倒伏 |
| 病害压力 | 无明显病害、轻微、中等、严重 |
| 病虫害类型 | 叶斑类病害、锈病、茎腐病、穗腐病、玉米螟等 |

成熟期如果填写倒伏率和穗腐率，系统会在未手动选择等级时自动生成：

- 倒伏等级。
- 病害压力。

## 3. 关键指标优先展示

公开示范阶段会从全量数据中提取“关键指标卡”，放在普通文字摘要和完整数据前面。

不同阶段的关键指标不同，例如：

| 阶段 | 关键指标 |
|---|---|
| 出苗 | 出苗率、苗势、整齐度、异常情况、天气影响 |
| 拔节 | 长势、株型、整齐度、病害压力 |
| 抽雄吐丝 | 花期协调、吐丝整齐度、气象影响、天气影响 |
| 灌浆 | 灌浆进度、保绿性、病害压力、抗逆表现 |
| 成熟 | 倒伏率、倒伏等级、倒折率、穗腐率、病害压力、脱水速度 |
| 收获 | 实收亩产、含水率、倒伏影响、商品性 |

## 4. 阶段照片自动优先展示

示范点详情页和阶段传播页继续遵循：

```text
阶段照片 / 视频
    ↓
关键指标卡
    ↓
公开摘要
    ↓
完整数据
```

示范点列表卡片也优先使用最新公开阶段照片；如果该示范点没有公开阶段照片，再使用后台设置的示范点封面。

未审核照片、发布后新增但未重新审核的照片不会公开展示。

## 5. 示范点排序

示范点公开列表不再显示“默认排序 / 按热度 / 按距离”按钮，减少手机端操作负担。

页面默认进入地图展示，并以“距离最近的示范点”为核心体验：

| 场景 | 展示规则 |
|---|---|
| 用户授权定位 | 前端自动带 `lat/lng` 重新进入页面，后端按直线距离从近到远排序 |
| URL 已带 `lat/lng` | 直接按距离排序，并在列表卡片展示“距您约 X 公里” |
| 用户拒绝定位或浏览器不支持定位 | 继续按后台维护的展示顺序作为兜底 |

距离排序支持参数：

```text
/sites/?lat=34.750100&lng=113.620100
```

手机端不再需要点击“按距离”，页面会自动尝试调用浏览器定位；定位失败时不阻塞用户浏览。

## 6. 地图弹窗增强

高德地图弹窗展示：

- 示范点名称。
- 距离。
- 当前阶段。
- 热度。
- 缩略图。
- 查看详情入口。

地图仍然使用高德地图，不引入国外地图服务。

## 7. 移动端紧凑与性能优化

本 Sprint 做了以下移动端优化：

- 关键指标卡在移动端使用两列紧凑布局。
- 示范点阶段记录缩小间距和字号。
- 非首屏图片使用懒加载。
- 阶段图片增加异步解码。
- 静态资源版本更新，避免旧 CSS/JS 缓存。

## 8. 部署说明

本 Sprint 没有新增数据库迁移。

运行环境部署后需要执行：

```powershell
python manage.py collectstatic --noinput
```

如果使用迁移包重新部署代码，确保 `static/css/site.css` 和 `static/js/site-map.js` 已同步。

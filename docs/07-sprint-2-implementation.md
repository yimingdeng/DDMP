## 3. 数据模型

### 品种

保存名称、URL 标识、定位、简介、审定信息、适宜区域、熟期、株穗粒型、密度、栽培建议、风险提示、推荐状态、排序及内部备注。

发布时强制校验一句话定位和品种简介；密度上限不得小于下限。

### 核心卖点

保存所属品种、标题、类型、简述、详情、展示数据说明、排序及内部依据。每个品种内 URL 标识唯一。

### 示范点

保存所属品种、区域、省市县、乡镇/详细地址公开开关、面积、播种日期、种植密度、当前阶段、主要表现、参观状态、排序和内部信息。

### 联系人

联系人可以关联多个示范点。公开姓名和公开电话分别控制；后台列表默认脱敏手机号。

## 4. 发布规则

- 草稿内容不能通过公开 URL 查看。
- 已发布内容进入首页、列表和详情。
- 已归档内容保留后台数据，但从公开页面隐藏。
- 示范点所属品种未发布时，示范点也不会公开。
- 后台支持批量发布、归档和退回草稿。
- 发布动作记录时间和后台操作人。

## 7. 当前限制

- 本机开发仍使用 SQLite 兜底；共享测试和正式环境仍按方案使用 PostgreSQL。
- 图片、视频和封面管理属于 Sprint 3，因此当前使用 CSS 视觉占位。
- 后台账号需要项目负责人使用 `createsuperuser` 创建，不自动生成默认密码。
- 精细的内容管理员和线索处理人员角色将在对应模块完成时配置。
- 应用内浏览器自动控制组件本轮无法连接，但真实 HTTP 和测试客户端验证均已完成。

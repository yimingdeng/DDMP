# 区域联系人与咨询线索实施记录

## 1. 功能闭环

本轮完成：

> 后台维护区域联系人并发布 → 全站公开展示区域经理 → 用户填写联系方式和需求 → 后台查看、分配并更新跟进状态

原有“示范点联系人”继续用于具体示范点的看田服务；新增“区域联系人”用于河南、山东等区域的统一业务服务，两类联系人互不混用。

## 2. 后台使用

登录 `/admin/` 后可看到“区域服务与咨询”：

### 区域联系人

填写：

- 服务区域，例如“河南”“山东”。
- 联系人姓名。
- 职位，例如“区域经理”。
- 联系电话。
- 可选的公开服务说明。
- 排序和发布状态。

只有“已发布”的区域联系人会出现在公开页面。数值较小的排序优先展示。

### 咨询记录

用户提交后自动产生一条“待处理”记录。后台可以：

- 查看姓名、电话、所在区域、单位、咨询内容和来源页面。
- 将线索分配给某位区域联系人。
- 填写内部跟进记录。
- 设置为“跟进中”“已完成”或“无效线索”。
- 使用列表批量操作更新处理状态。

每次后台修改状态、负责人或内部记录时，系统记录最近跟进时间和操作人。

## 3. 前台展示与提交

- 所有公开页面底部均显示“区域服务”和“我要咨询”。
- 区域经理电话支持手机直接拨打。
- 用户填写姓名、联系电话、所在区域、单位和咨询内容。
- 用户必须明确勾选同意保存信息并接受联系。
- 提交成功或失败后显示明确提示，并返回原页面联系区域。
- 所在区域支持从已发布区域中选择，也允许用户输入其他地区。

## 4. 数据和安全规则

- 咨询记录不会出现在公开页面，只能由有权限的后台账号查看。
- 联系电话只允许数字、空格、`+`、`-` 和括号，长度为 7—30 位。
- 表单启用 Django CSRF 防护。
- 返回地址只允许当前站点，防止外部跳转。
- 使用隐藏字段拦截基础机器人提交。
- 后台列表默认对联系电话进行部分脱敏，详情页可查看完整信息用于业务跟进。

测试和正式环境应按账号职责限制咨询记录权限，并定期清理不再需要的个人信息。

## 2. 后台使用方法

登录 `http://127.0.0.1:8000/admin/`，进入品种、核心卖点或示范点编辑页，在“媒体素材”区域新增内容。

### 上传图片

1. 媒体类型选择“图片”。
2. 填写标题、说明和替代文本，选择图片文件。
3. 需要作为页面主图时勾选“设为封面”。
4. 使用“排序”控制展示顺序，数值越小越靠前。
5. 状态设为“已发布”后保存。

发布图片必须填写替代文本。图片支持 JPEG、PNG、WebP，单张不超过 10 MB，像素总量不超过 4000 万。

### 登记视频链接

1. 媒体类型选择“视频链接”。
2. 选择抖音、微信视频号或其他平台。
3. 填写平台提供的 `https://` 或 `http://` 公开访问链接。
4. 可上传视频封面；视频作为页面封面时必须上传封面。
5. 设置排序和“已发布”状态后保存。

平台当前保存视频链接，不下载、不转码，也不直接托管第三方视频。微信或抖音链接能否在外部打开，取决于对应平台的链接有效性和访问策略。

### 上传本地视频

1. 媒体类型选择“本地视频”。
2. 在“本地 MP4 视频”字段选择 `.mp4` 文件。
3. 建议视频使用 H.264 视频编码和 AAC 音频编码，以兼容微信、Safari 和主流安卓浏览器。
4. 可上传 JPEG、PNG 或 WebP 视频封面；如设为首页封面则必须上传视频封面。
5. “视频平台”“视频链接”和普通“图片”字段保持为空。
6. 设置排序和“已发布”状态后保存。

单个本地视频最大 200 MB。系统不执行视频转码，编码不兼容的视频需要在上传前转换为 H.264 + AAC 的 MP4。

## 3. 展示规则

- 对象存在已发布封面时，封面优先用于首页、列表和详情摘要区域。
- 没有明确封面时，第一张可展示的已发布媒体作为视觉候选。
- 所有已发布图片按“排序、创建时间”进入详情画廊。
- 点击画廊图片可查看原图，并支持上一张、下一张和键盘方向键。
- 外链视频在详情页显示为独立卡片，点击后跳转对应平台。
- 本地视频在首页和详情页使用 HTML5 原生播放器，支持手机端行内播放。
- 对象没有媒体时保持原有页面结构，不显示空画廊或空视频模块。

## 4. 安全与存储

- 不信任文件扩展名，使用 Pillow 读取并验证图片真实格式。
- 拒绝损坏图片、不支持格式、超过 10 MB 或超过 4000 万像素的图片。
- 视频链接只允许 HTTP/HTTPS 协议。
- 本地视频只允许 `.mp4`，最大 200 MB，并检查 MP4 容器签名以拒绝改名伪装文件。
- 本地视频记录文件大小、MIME 类型和 SHA-256 摘要。
- 原图和缩略图保存在 `MEDIA_ROOT`；开发环境默认是 `D:\DDMP\.local\media`。
- 替换或删除媒体时同步删除不再使用的文件。
- 测试和运行环境必须为媒体目录安排独立、持久化存储和备份。

## 7. 当前限制

- 本地视频不提供自动转码、多清晰度或第三方平台自动同步。
- 开发环境适合功能测试；正式环境应由 Caddy 直接提供媒体文件并为 `MEDIA_ROOT` 安排容量监控和备份。
- 自动缩略图用于页面加载优化，原图仍需控制分辨率和内容质量。
- 后台内联表单会同时显示图片和视频字段，录入人员需按媒体类型填写对应字段。
- 手机微信、抖音内置浏览器的最终兼容性应在具有 HTTPS 域名的测试环境进行真机验收。

## 8. 封面冲突提示优化（2026-07-01）

- 错误信息明确说明同一品种、核心卖点或示范点只能保留一张封面。
- 单独将一条媒体素材设为封面时，系统仍会自动取消原封面。
- “图片”和“视频封面”字段增加用途说明，避免视频链接误传到普通图片字段。

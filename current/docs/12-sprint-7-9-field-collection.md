# 第二阶段 Sprint 7–9：负责人田间采集

## 1. 已实现范围

### Sprint 7：负责人权限和示范点绑定

- 使用 Django 个人账号登录负责人端。
- 管理员在“负责人示范点分配”中绑定账号和示范点。
- 负责人只能看到有效绑定的示范点；直接修改 URL 也不能访问其他示范点。
- 超级管理员可以从负责人端查看全部示范点，便于支持和验收。
- 公开展示端、负责人采集端和后台管理端相互隔离。

负责人入口：`/collection/login/`。

### Sprint 8：任务、草稿、提交和历史

- “我的示范点”显示七阶段总体进度。
- 示范点任务页显示播种、出苗、拔节、抽雄吐丝、灌浆、成熟和收获七个阶段。
- 阶段记录可保存不完整草稿，草稿不进入公开展示。
- 正式提交时执行完整校验；提交后负责人端锁定记录。
- 每次保存草稿和正式提交都记录操作人、时间和动作。
- 示范点任务页展示最近操作历史。

### Sprint 9：七阶段表单和现场照片

- 七阶段采用各自的日期、数值、百分比、评价和文字字段。
- 百分比限制为 0–100，数值不允许负数。
- 正式提交时每个阶段至少需要一张现场照片。
- 支持手机拍照或相册多选上传 JPEG、PNG、WebP。
- 上传时自动纠正照片方向、压缩为 JPEG，最长边不超过 1920 像素。
- 草稿照片可以删除；正式提交后的照片锁定。
- 当前不保存 GPS 和 EXIF 信息。
- 收获期保存实收面积、重量和含水率原始值；正式亩产公式确认前不自动计算。

### 手机采集优化

- 固定业务字段优先使用下拉单选或多选按钮，减少负责人手工输入。
- 仅日期、田间测量数值、种子批次、肥料用量和补充评价保留手工输入。
- 照片提供“拍照/继续拍照”和“从相册选择照片”两个入口；手机端拍照默认调用后置相机，同一阶段可连续拍摄多张。
- 提供“摄像/继续摄像”入口，同一阶段可上传多段 MP4、MOV 或 WebM 现场视频，单个文件最大 200 MB。
- 输入播种密度和行距后，自动计算株距并保留一位小数。
- 填写抽雄日期和吐丝日期时，自动计算花期间隔天数。
- 填写实收面积和实收重量时，自动计算未经标准水分折算的实收亩产。
- 自动计算在浏览器中即时预览，并由服务端重新计算后保存，避免篡改前端结果。
- 负责人密码只要求至少 6 位，不要求大小写、符号或非纯数字。
- 登录页自动记住上次用户名；密码通过浏览器/手机系统密码管理器安全填充，不以明文保存在网页本地。

## 2. 管理员配置步骤

1. 打开后台“认证和授权 → 用户”，为每位负责人创建独立账号。
2. 姓名建议填写在用户的姓、名字段中；不要多人共用账号。
3. 打开“田间采集 → 负责人示范点分配”。
4. 选择负责人和示范点，保持“有效”勾选后保存。
5. 也可以进入示范点详情，在“负责人示范点分配”内联区域完成绑定。
6. 将 `/collection/login/` 和账号密码单独发给负责人，首次登录后安排修改密码。

停用负责人访问时，取消分配记录的“有效”即可；不要删除历史采集数据。

## 3. 负责人操作步骤

1. 手机打开 `/collection/login/`，使用个人账号登录。
2. 在“我的示范点”选择负责的示范点。
3. 根据田间进度进入相应阶段。
4. 数据不完整时点击“保存草稿”。
5. 补齐带 `*` 字段并上传至少一张现场照片。
6. 确认无误后点击“提交阶段记录”。
7. 提交后如需修改，应等待后续审核模块提供正式退回流程，避免绕过历史记录。

## 4. 数据和权限设计

- `SiteAssignment`：负责人和示范点的授权关系。
- `Observation`：一个示范点每个阶段一条当前采集记录。
- `ObservationPhoto`：阶段现场照片和说明。
- `CollectionEvent`：不可由负责人编辑的草稿/提交历史。
- 七阶段差异字段保存在 JSON 数据中，公共身份、状态、操作者和时间使用固定数据库字段。
- 这种结构允许试运行后调整阶段字段，同时不牺牲权限、状态和审计查询。

## 5. 验收清单

- [ ] 未登录访问采集端会跳转登录页。
- [ ] 负责人只能看到绑定给自己的示范点。
- [ ] 修改 URL 不能查看其他负责人的示范点。
- [ ] 七个阶段均可保存草稿。
- [ ] 草稿可在下次登录后继续编辑。
- [ ] 缺少必填字段或现场照片时不能正式提交。
- [ ] 手机上传大照片后自动压缩。
- [ ] 正式提交后负责人不能修改或删除照片。
- [ ] 后台可以查看采集数据、照片和操作历史。

# Sprint 5 实施记录：抖音适配、线索和统计

## 1. 本 Sprint 包含的功能

### 抖音与移动端适配

- 支持全面屏安全区域，避免顶部和底部被系统控件遮挡。
- 抖音内置浏览器采用更稳定的非吸顶页头。
- 表单控件使用移动端友好的字号和触控高度，避免 iPhone 输入时自动缩放。
- 本地视频继续使用网页原生播放和 `playsinline`，图片、电话链接和表单无需跳出浏览器。
- 增加键盘焦点、跳过导航、表单标签和减少动画支持。

### 线索安全与跟进

- 提交按钮防连点，提交中显示明确状态。
- 同一会话 10 分钟最多提交 5 次，同一 IP 的匿名哈希维度最多 20 次。
- 服务端保留原有蜜罐字段和 UUID 防重复提交。
- 校验失败或触发限流后，姓名、电话、地区、单位、意向和咨询内容自动回填；隐私同意项不自动勾选。
- 后台支持按状态、意向、来源、地区、品种、示范点和时间筛选。
- 新增不可直接修改或删除的跟进历史：处理状态、跟进内容、下一步计划、记录人和时间。
- 查看完整线索、修改线索和修改站点配置会写入独立审计记录，审计摘要不保存手机号等敏感内容。

### 访问与线索统计

- 使用随机 Cookie 生成匿名访客标识；数据库仅保存 SHA-256 哈希，不使用设备指纹。
- 记录公开 HTML 页面访问，排除后台、静态文件、媒体、健康检查、二维码跳转和表单提交地址。
- 后台“访问与线索统计 → 统计工作台”显示：
  - 7 日 PV、30 日 PV、30 日估算 UV。
  - 已发布品种、示范点、媒体数量和待处理线索数。
  - 每日访问趋势、热门页面、访问来源。
  - 线索来源、状态和意向分布。
  - 最近线索及内容管理快捷入口。

## 2. 隐私说明

公开地址为 `/privacy/`，页脚和咨询同意项均有入口。页面说明收集范围、使用目的、保存与保护、用户权利和联系方式，并显示后台配置的隐私版本。每条线索继续保存提交时的隐私版本与同意时间。

## 3. 后台使用方法

1. 打开“访问与线索统计 → 统计工作台”查看运营数据。
2. 打开“区域服务与咨询 → 咨询记录”，按来源、意向、地区或内容对象筛选。
3. 进入线索详情后，在页面底部新增一条跟进记录；保存后线索状态同步更新。
4. 打开“基础管理 → 敏感操作审计”查询线索查看和修改记录。

## 4. 数据口径与限制

- PV 是成功返回的公开 HTML 页面访问次数。
- UV 是 30 天内匿名随机标识去重数；清除 Cookie 或更换浏览器会被视为新访客，因此称为“估算 UV”。
- IP 只在请求当下用于限流并立即转成不可逆哈希键，不写入业务数据库。
- Windows 局域网 HTTP 可做功能测试；抖音投放前仍需使用正式 HTTPS 域名，在 Android 和 iPhone 的抖音内置浏览器各完成一次真机验收。
- 线索导出、销售漏斗转化率和外部 CRM 对接不在本 Sprint 范围内。

# Sprint 4 实施记录：传播和留资

## 1. 实施结果

Sprint 4 已形成可追踪的传播闭环：

> 后台生成带来源的二维码 → 用户扫码进入品种/示范点/咨询页 → 来源在会话中保留 → 用户提交具体意向 → 后台查看来源、目标对象和处理状态

本轮在已完成的基础咨询功能上新增：

- 微信及常规社交平台可读取的 Open Graph 分享标题、描述、图片和规范地址。
- 站点默认分享标题、描述、分享图及公开访问地址配置。
- 首页、品种、卖点和示范点的动态分享信息。
- 首页、品种详情、示范点详情和咨询表单四类二维码目标。
- 二维码 PNG 预览与下载、启用/停用、用途说明和来源代码。
- 扫码次数与最近扫码时间。
- 来源代码和首次落地页在会话内保留。
- 产品咨询、预约看田、申请试种、代理合作四类意向。
- 线索自动关联当前品种或示范点。
- 隐私说明版本和用户同意时间记录。
- 防重复提交标识；重复刷新不会创建重复线索。

## 2. 站点分享配置

进入后台“站点配置”，设置：

- 默认分享标题。
- 默认分享描述。
- 默认分享图，建议 1200×630 横图。
- 公开访问地址，例如正式环境的 `https://show.example.com`。
- 咨询隐私说明和版本号。

首页有推荐品种时优先使用推荐品种名称、定位和封面；品种与示范点详情优先使用各自内容。缺少专属图片时回退到默认分享图。

当前完成的是网页分享元数据。微信 JS-SDK 的自定义分享回调仍需要认证公众号、正式 HTTPS 域名和平台密钥，具备这些条件后再接入。

## 3. 渠道二维码

进入后台“传播与二维码 → 渠道二维码”新增记录：

1. 填写名称和用途。
2. 选择目标：平台首页、品种详情、示范点详情或咨询表单。
3. 品种/示范点目标需选择对应对象。
4. 填写来源代码，例如 `wechat_moments`、`douyin_profile`、`henan_field_sign`。
5. 保存后可预览或下载 PNG。

来源代码仅允许小写字母、数字、下划线和连字符，长度为 2—40 位，并且不能重复。

开发联调时，站点“公开访问地址”已配置为 `http://192.168.1.232:8000`。开发机 IP 变化后需要同步更新；正式投放前必须改成正式 HTTPS 域名并重新下载二维码。

停用二维码后，扫码会显示提示并返回首页；已归档或未发布的目标也不会暴露详情。历史二维码记录和扫码次数保留。

## 4. 留资与来源

咨询表单现在保存：

- 姓名、联系电话、所在区域、单位和咨询内容。
- 产品咨询、预约看田、申请试种或代理合作。
- 感兴趣的品种和示范点。
- 来源代码和首次落地页。
- 隐私说明版本、同意状态和同意时间。
- 防重复提交标识。

从示范点详情提交时自动关联该示范点和品种。示范点未开放看田时，前台禁用“预约看田”，服务端也会再次校验。

后台咨询列表可按状态、意向类型、来源、区域和时间筛选，并继续支持分配区域经理和更新跟进状态。
